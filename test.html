I'm very new snowflake database. Could anyone pls help me out to find the right typecast conversion function that can be used

Hello Experts,

 

I'm very new snowflake database. Could anyone pls help me out to find the right typecast conversion function that can be used for the below mentioned load scenario

 

I want to load the below mentioned timestamp data into the snowflake table

 "eventdate": "7/5/2017 12:00:15 AM",

Target field is defined as TIMESTAMP_NTZ(9). When I try to load the data using the below mentioned insert statement, I am getting the below error

 

INSERT INTO TARGET_TABLE_NAME

SELECT

(TO_TIMESTAMP(DATALOAD:eventdate)::TIMESTAMP_NTZ(9)

FROM

STAGE_TABLE_NAME

 

Error I'm getting: Timestamp '7/5/2017 08:00:15 PM' is not recognized

 

when I try to convert this format using below query, I'm getting the desired result

 

select to_timestamp('7/5/2017 08:00:15 PM','mm/dd/yyyy hh12:mi:ss am')

Output - 2017-07-05 20:00:15.000

 

Since I am using hard coded timestamp value and trying to convert it to 24 hours format, I'm am able to convert but When I use the same Approach in the insert statement as mentioned below, I m getting error too many arguments error.

 

INSERT INTO TARGET_TABLE_NAME

SELECT

(TO_TIMESTAMP(DATALOAD:eventdate),'dd/mm/yyyy hh12:mi:ss am')::TIMESTAMP_NTZ(9)

FROM

STAGE_TABLE_NAME

 

My stage table data is in json format. So we are using DATALOAD keyword in the insert statement to read the field from the json and mapping into the target table.

 

Any help is highly appreciated.

 

Regards,

Dinesh M

    Hi Dinesh, you got it almost right but it looks like the parentheses are misplaced and that's why you're getting an error. Try this (reformatted a bit to make it clearer):

     

    INSERT INTO TARGET_TABLE_NAME

    SELECT

    TO_TIMESTAMP(

    DATALOAD:eventdate,

    'dd/mm/yyyy hh12:mi:ss am'

    )::TIMESTAMP_NTZ(9)

    FROM STAGE_TABLE_NAME

     

    In other words, TO_TIMESTAMP can take 2 arguments, but the way you had written it, the format string 'dd/mm/yyyy hh12:mi:ss am' was not passed to it, as it was outside of the parentheses.
